Your best bet is to use a pre-packaged setup for your operating system. Newer Linux distributions may have it available via the standard package repositories, so check that first! Known public packaged builds:

  • Debian Linux: http://packages.debian.org/gdnsd
  • Ubuntu Linux: https://launchpad.net/ubuntu/+source/gdnsd/
  • FreeBSD: http://portsmon.freebsd.org/portoverview.py?category=dns&portname=gdnsd
  • RPM-based distros: The best I can google up lately is Matthias's packages here: http://dl.marmotte.net/rpms/redhat/el6/x86_64/gdnsd-1.8.3-1.el6/. I used to maintain an upstream specfile (which was also based on Matthias's specfile), and you could use the last version of that as a starting point as well: https://github.com/blblack/gdnsd/blob/v1.9.0/pkg/rpm/gdnsd.spec
  • Raw Source Tarballs: http://downloads.gdnsd.org/ - The tarball URLs here are hosted on S3 and should be stable as packaging sources. If you're building from scratch for a one-off installation, you may find the tarball version easier to work with than a git checkout (fewer dependencies).
